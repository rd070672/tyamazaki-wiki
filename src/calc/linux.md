## 1. Ubuntu / Linux の基本的な概念

### 1.1 Linux とディストリビューション

- **Linux**
  - OS の「カーネル（核）」の名称。
  - 実際には、カーネル＋周辺ツール一式をまとめて「Linux」と呼ぶことも多い。

- **ディストリビューション（distribution, distro）**
  - Linux カーネルに、  
    - パッケージ管理の仕組み  
    - 各種ツール・アプリ  
    を組み合わせた「セット商品」。
  - 例：Ubuntu, Debian, Fedora, Arch Linux など。

- **Ubuntu**
  - Debian 系ディストリビューション。
  - デスクトップ用途・開発用途・サーバー用途で広く使われる。
  - パッケージ管理に `apt` を使う。

> イメージ：  
> 「Linux = エンジン」  
> 「Ubuntu = エンジン＋タイヤ＋ハンドル＋メーターが付いた完成車」。

---

### 1.2 シェルとターミナル

- シェル（shell）
  - キーボードからのコマンドを受け取り、OS に処理をお願いする「窓口プログラム」。
  - 代表例：`bash`, `zsh` など。
- ターミナル（terminal, 端末）
  - シェルを表示・操作するための「画面（アプリ）」。
  - Ubuntu では「ターミナル」アプリを起動すると、その中で `bash` が動いていることが多い。

整理すると：

- シェル：中身（コマンドを解釈するプログラム）
- ターミナル：見た目（黒い画面のアプリ）

## 2. ターミナルと基本操作の考え方

ここでは、コマンドを「丸暗記する」というより、

- 何をしたいときに
- どの種類のコマンドを使うか

という役割ベースで整理します。

### 2.1 ターミナルの役割

- Windows のエクスプローラや macOS の Finder に相当する操作を、  
  「文字だけで行う」ためのインターフェース。
- ファイルの一覧表示、移動、コピー、削除などをすべてキーボードだけで行える。

---

### 2.2 基本コマンドの“役割マップ”

代表的なコマンドを、「何をしたいか」で分類すると次のようになる。

| やりたいこと                     | 代表的なコマンド（名前だけ） |
|----------------------------------|------------------------------|
| 今どこにいるか知りたい           | `pwd`                        |
| ファイルやフォルダを一覧したい   | `ls`                         |
| 別のフォルダに移動したい         | `cd`                         |
| 新しいフォルダを作りたい         | `mkdir`                      |
| ファイルやフォルダをコピーしたい | `cp`                         |
| ファイルやフォルダを移動したい   | `mv`                         |
| ファイルやフォルダを削除したい   | `rm`                         |

**考え方のポイント：**

- `ls`：**List**  
  → 「中身を見せて」
- `cd`：**Change Directory**  
  → 「場所を変える」
- `mkdir`：**Make Directory**  
  → 「フォルダを作る」
- `cp`：**Copy**  
  → 「コピーする」
- `mv`：**Move**  
  → 「移動（＋名前変更）する」
- `rm`：**Remove**  
  → 「消す」

> 注意点：  
> `rm` は「ゴミ箱」ではなく、基本的に即消える。  
> GUI の削除より「重い操作」であることを意識して使う。

## 3. `apt` によるパッケージ管理のイメージ

Ubuntu では、ソフトウェアのインストール・更新・アンインストールを  
`apt` というパッケージマネージャーで行う。

### 3.1 `apt` の役割

- 「アプリのインストール用アプリ」のようなもの。
- ソフトウェアを
  - 探す
  - 導入する
  - 更新する
  - 削除する
  までを、一括で管理してくれる。

### 3.2 よく行う操作（概念だけ）

1. **パッケージリストの更新**
   - 「世の中で今提供されている最新版は何か？」というカタログを更新する。
   - 対応するのが `apt update`。

2. **システムの更新**
   - 実際にインストール済みのソフトを最新にする。
   - 対応するのが `apt upgrade`。

3. **新しいソフトウェアを入れる**
   - 例：`htop` というモニタリングツールを入れたいときは、`apt install htop` に相当する操作を行う。

4. **いらなくなったソフトウェアを消す**
   - `apt remove` で不要なパッケージを削除できる。

> ポイント：  
> - 基本的に、アプリはブラウザで探してダウンロードするのではなく、`apt` 経由で入れる  
> - その方が、更新も削除も管理しやすい（依存関係もよしなに処理してくれる）

## 4. 権限と `sudo` の基本的な考え方

### 4.1 Linux の権限モデル（ざっくり）

Linux では、ファイルやフォルダに対して

- 誰が
- どのように（読む・書く・実行する）

ことができるかを細かく制御している。

`ls -l` で表示される

- `-rw-r--r--`
- `drwxr-xr-x`

のような文字列が、各ファイルの「権限」を表す。

#### 権限の内訳

- 先頭文字：
  - `-` : 通常ファイル
  - `d` : ディレクトリ
- 続く 9 文字を 3 文字ずつに区切る：
  1. 所有者（user）の権限
  2. グループ（group）の権限
  3. その他（others）の権限

各 3 文字は `r`（read）、`w`（write）、`x`（execute）から構成される。

例：`-rw-r--r--`

- 所有者：`rw-` → 読み書き可
- グループ：`r--` → 読み取りのみ
- その他：`r--` → 読み取りのみ

> イメージ：  
> 「このファイルは、自分は編集してよいが、他の人には読むだけ許可する」といったルールを設定している。

---

### 4.2 `sudo` の役割と考え方

- 一般ユーザーには「勝手にシステム全体を書き換えられない」ように制限がある。
- しかし、パッケージのインストールや OS の設定変更など、  
  管理者として操作しないといけない場面が確実にある。

そこで使うのが **`sudo`**（“superuser do” の略）。

- 特定のコマンドだけ「管理者権限で実行する」ための仕組み。
- Ubuntu では、パッケージ管理（`apt`）やシステム設定の変更を行う際に頻繁に使う。

#### `sudo` を使うときの考え方

- 「システム全体に影響する操作をしているか？」を一瞬考える。
- もしそうなら、`sudo` が必要な可能性が高い。
- 必要もないのに何でも `sudo` を付けると、
  - 不要に大きな権限でファイルを作ってしまう
  - 後から権限で詰む  
  といったトラブルになりがち。

> 目安：  
> - `apt` でソフトを入れる・更新する → `sudo` 必須  
> - 自分のホームディレクトリの中だけで作業 → 基本的に `sudo` は不要
